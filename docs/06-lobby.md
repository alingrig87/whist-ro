# Commit 06 — Lobby: Create & Join Tables

## 🇬🇧 English

### Requirements

- Authenticated users see a list of open tables
- Create a table with a name + max players setting
- Join any open table (status: 'waiting', not full)
- Join via direct link: `/table/:tableId`
- Leave a table before the game starts

### What Was Implemented

| File | Purpose |
|------|---------|
| `src/components/Lobby/LobbyPage.tsx` | Main lobby with table list |
| `src/components/Lobby/CreateTableModal.tsx` | Modal to configure new table |
| `src/components/Lobby/TableCard.tsx` | Single table preview card |
| `src/lib/tables.ts` | Firestore CRUD for tables |

### Table Lifecycle

```
create → 'waiting' ──(all ready + host starts)──→ 'playing' ──(last round done)──→ 'finished'
              ↑
         players join/leave freely
```

A table stays in `'waiting'` until the host clicks **"Start Game"**, which requires:
- Minimum 3 players joined
- All joined players have `ready: true`

### `createTable` Function

```typescript
async function createTable(
  name: string,
  maxPlayers: number,
  creatorUid: string,
  groupId?: string
): Promise<string> {
  const ref = doc(collection(db, 'tables'))
  await setDoc(ref, {
    name,
    createdBy: creatorUid,
    createdAt: serverTimestamp(),
    status: 'waiting',
    playerOrder: [],
    maxPlayers,
    scores: {},
    currentRound: 0,
    totalRounds: 15,
    groupId: groupId ?? null,
  })
  // Also add creator as first player
  await setDoc(doc(db, 'tables', ref.id, 'players', creatorUid), {
    displayName: ...,
    photoURL: ...,
    ready: false,
    joinedAt: serverTimestamp(),
  })
  return ref.id
}
```

The table ID is auto-generated by Firestore (`doc(collection(...))` without an ID).
This gives URL-safe random IDs like `KmN9xPqR2vT...`.

### Real-Time Table List

The lobby subscribes to open tables with a Firestore query:

```typescript
query(
  collection(db, 'tables'),
  where('status', '==', 'waiting'),
  orderBy('createdAt', 'desc'),
  limit(20)
)
```

`onSnapshot` fires whenever a table is created, joined, or removed — the list
updates live without any refresh.

### Why `limit(20)`

Without a limit, adding the 21st table would re-send all 21 documents to every
lobby client. With `limit(20)`, old full/finished tables naturally drop off.

### Optimistic UI for Join

When a user clicks "Join", we update the UI immediately (add them to local state)
and fire the Firestore write in the background. If the write fails (race condition —
table just filled up), we revert with an error toast.

## 🇷🇴 Română

### Cerințe

- Utilizatorii autentificați văd o listă de mese deschise
- Creează o masă cu nume + număr maxim de jucători
- Alătură-te oricărei mese deschise
- Join prin link direct: `/table/:tableId`
- Pleacă de la masă înainte ca jocul să înceapă

### Ce s-a implementat

| Fișier | Scop |
|--------|------|
| `src/components/Lobby/LobbyPage.tsx` | Lobby-ul principal cu lista meselor |
| `src/components/Lobby/CreateTableModal.tsx` | Modal pentru configurarea mesei noi |
| `src/components/Lobby/TableCard.tsx` | Previzualizare masă individuală |
| `src/lib/tables.ts` | CRUD Firestore pentru mese |

### Ciclul de viață al mesei

```
creare → 'waiting' ──(toți ready + host pornește)──→ 'playing' ──(ultima rundă)──→ 'finished'
```

O masă rămâne în `'waiting'` până când host-ul apasă **"Pornește jocul"**, care necesită:
- Minimum 3 jucători alăturați
- Toți jucătorii alăturați au `ready: true`

### Listă de mese în timp real

Lobby-ul se abonează la mesele deschise cu o interogare Firestore:

```typescript
query(
  collection(db, 'tables'),
  where('status', '==', 'waiting'),
  orderBy('createdAt', 'desc'),
  limit(20)
)
```

`onSnapshot` se declanșează ori de câte ori o masă e creată, se alătură cineva sau
e ștearsă — lista se actualizează live fără refresh.

### UI optimist pentru join

La click pe "Join", actualizăm UI-ul imediat (adăugăm jucătorul în starea locală)
și trimitem write-ul Firestore în background. Dacă write-ul eșuează (condiție de
cursă — masa tocmai s-a umplut), revenim cu un mesaj de eroare.
